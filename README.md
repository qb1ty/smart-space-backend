# 🏢 Smart Space API — Enterprise B2B Workspace Booking Backend

**Smart Space API** — это высоконагруженный бэкенд для системы бронирования рабочих мест, переговорок и зон отдыха (PlayStation Lounge) в двухэтажном интерактивном 3D-офисе. 

Проект спроектирован по принципам чистой архитектуры и Enterprise-разработки, делая упор на безопасность параллельных вычислений, защиту финансовых транзакций (баланс монет) и глубокую бизнес-аналитику.

---

## ⚡ Ключевые архитектурные решения и фичи

* 🛡️ **Защита от Race Condition (Двойного бронирования):** Используются распределенные блокировки (Distributed Locks) на базе **Redis** (`acquireLock` / `releaseLock`), гарантирующие, что при одновременном клике двух пользователей на один стол бронь достанется только первому.
* 🔒 **ACID-транзакции и атомарность:** Вся работа с финансами (списание и возврат баланса монет), проверка пересечений времени (`checkOverlappingBookings`) и каскадная отмена будущих броней при увольнении выполняются строго внутри изолированных транзакций `Prisma.$transaction`.
* 👮 **Безопасная аутентификация и сессии:** 
  * Сессионная аутентификация через `express-session` и `withCredentials` с хранением в Redis.
  * Глобальная типизация сессий через `declare module "express-session"`.
  * Декларативное разграничение прав через `AuthGuard` и `AdminGuard` (Role-Based Access Control).
  * Механизм **Soft Ban (`isActive`)** с проверкой на уровне базы данных при совершении транзакций.
* 📊 **Highload Аналитическое ядро (`ReportsModule`):** 
  * Расчет эффективности использования офиса: **Occupancy Rate** (процент загруженности пространств) и **No-Show / Cancellation Rate** (статистика отмен сотрудников).
  * Выполнение тяжелых агрегирующих запросов (`groupBy`, `_sum`, `_count`) параллельно в одном транзакционном потоке без перегрузки Event Loop.
* 🧼 **Clean Code & FSD-friendly API:** Отсутствие «толстых» сервисов — вся математика, парсинг дат и SQL-запросы вынесены в чистые функциональные хелперы и утилиты. Гарантированная защита от утечки чувствительных данных (хэшей паролей) через единый словарь `SAFE_USER_SELECT`.

## 🛠 Технологический стек

| Категория | Технологии |
| :--- | :--- |
| **Ядро** | Node.js, NestJS, TypeScript, RxJS |
| **База данных & ORM** | PostgreSQL, Prisma ORM |
| **Кэш & Блокировки** | Redis, `@nestjs/cache-manager` |
| **Аутентификация** | Express Session, bcrypt, Custom Guards & Interceptors |
| **Валидация** | Class-Validator, Class-Transformer, DTO |
